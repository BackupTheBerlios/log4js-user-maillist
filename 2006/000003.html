> > It should be simpler.
> > 
> > 
> > Let's take the standard Log4J's DTD
> (http://logging.apache.org/log4j/docs/api/org/apache/log4j/xml/log4j.dtd)
> > 
> > Now our imaginary application creates this log:
> > 
> > <log4j:eventSet>
> > 
> >   <log4j:event logger="com.foo.Xxxx" level="debug" ... />
> > 
> >   ...
> > 
> > </log4j:eventSet>
> > 
> > 
> > The logging framework should accumulate those logging event into some
> DOM
> > object. When reaching a specified threshold the contents of that DOM 
> > object are sent to a PUT-enabled HTTP server, which stores the contents 
> > in an XML file. That's it!
> > 
> > Then someone can use existing tools to search/analyze the logs or he can
> > write some XSL transformation to have the logs in his format.
> >
